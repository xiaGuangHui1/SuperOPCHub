## 一、项目概述

SuperOPCHub 是一个基于大语言模型（LLM）和 AI Agent 技术的智能匹配平台。它让**甲方用自然语言描述遇到的困难**，AI 自动拆解需求、分析画像、执行多维语义匹配、生成可解释的推荐理由——直接把最合适的人选推送到面前。

核心价值主张：让 OPC（One-Person Company / 一人公司）不再花时间「被找到」，释放创造力价值。

早上醒来，SuperOPCHub 已完成今天的匹配。作为 AI 中介，它用我的技能画像、偏好数据和历史评价，在我睡觉时自动对接了全球机会——省去了所有找客户、谈价格、签合同的环节。

我是一家 OPC。没有老板，没有员工。AI 代理替我承担了法务、财务、客服、排期——这些传统公司需要多个部门的职能。OPC 意味着组织简化到极致，决策权完全回归个人。

作为数字游民，我的办公室在任何地方。今天在大理，下周可以在清迈或里斯本。互联网让我在任何坐标接入全球工作网络，地域不再是职业限制。

下午 SuperOPCHub 推送了新的匹配通知和昨日结算报告。匹配、定价、签约、收款、信用管理——全自动运行。它把"找事做"变成了"选事做"。

傍晚走进一个陌生小镇的菜市场。这是环游世界的方式——不是存一年钱换一次度假，而是把世界变成日常活动范围。工作在任一坐标都可推进，生活在任一坐标都可展开。

#### 21:00 —— 墙已消失

晚上在车顶看星星。这种状态的终点是自由幸福——不是等财务自由才有资格自由，而是技术已经拆解了所有束缚：

### 3.2 关键实体与数据流

**甲方输入 → 需求结构体**

```
用户自然语言：
"我需要一个能做海外小众美妆品牌投放的操盘手，预算2万一个月"

        ↓ [需求解析 Agent]

结构化需求：
{
  "industry": "美妆/新消费",
  "channel": "TikTok",
  "skills": ["信息流投放", "KOL合作"],
  "budget": { "amount": 20000, "currency": "CNY", "period": "monthly" },
  "target": ["品牌声量", "带货转化"],
  "constraints": { "duration": "3个月起" },
  "pending_questions": ["偏品牌声量还是带货转化？", "有无对标账号？"]
}
```

**乙方上传 → 能力画像**

```
上传：简历 / LinkedIn / 作品集 / GitHub


        ↓ [乙方画像 Agent — 多模态解析]


动态能力画像：
{
  "hard_skills": {
    "TikTok信息流投放": { "level": "expert", "evidence": "花西子代运营", "projects": 12 },
    "KOL合作": { "level": "advanced", "evidence": "签约50+达人" }
  },
  "industry_knowledge": ["美妆个护", "新消费"],
  "work_style": {
    "autonomy": "high",
    "documentation": "well-documented"
  },
  "delivery": { "avg_cycle_days": 12, "on_time_rate": 1.0 },
  "soft_signals": { "english_proficiency": "business", "cross_border_exp": true }
}
```

**匹配引擎 → 推荐结果**

```
匹配维度对照：

硬技能匹配：       TikTok信息流投放  ←→  标签精确 + 同义词扩展
行业知识匹配：      美妆个护          ←→  向量语义相似度
```

## 六、评测标准

### 6.1 核心评测指标体系

#### 维度一：匹配质量（权重 35%）

| 指标 | 定义 | 目标值 | 测量方式 |
|---|---|---|---|
| 匹配意向转化率 | 甲方查看推荐后发起联系的占比 | ≥ 40% | 埋点统计：查看→联系 |
| 成交转化率 | 匹配推荐最终达成交易的占比 | ≥ 15% | 系统记录：匹配→签约 |
| 推荐相关性 (NDCG@5) | Top 5 推荐结果的排序质量 | NDCG@5 ≥ 0.75 | 基于实际签约反馈的标准化折损累计增益 |
| 甲方满意度评分 | 甲方对每次匹配的星级评分 | ≥ 4.2 / 5 | 交易完成后甲方打分 |

#### 维度二：匹配效率（权重 25%）

| 指标 | 定义 | 目标值 | 测量方式 |
|---|---|---|---|
| 需求到匹配时长 | 从需求发布到接收到首次推荐的耗时 | ≤ 60 分钟（Phase 1） | 系统时间戳差值 |
| 需求结构化准确率 | AI 解析的需求卡片与甲方真实意图的一致率 | ≥ 85% | 甲方确认率 + 人工抽检 |
| 平均成交周期 | 从需求发布到签约的全周期 | ≤ 48 小时 | 系统全链路时间戳 |
| 乙方画像更新延迟 | 交易完成后画像刷新的延迟 | ≤ 1 小时 | 系统监控 |

#### 维度三：用户体验（权重 20%）

| 指标 | 定义 | 目标值 | 测量方式 |
|---|---|---|---|
| 甲方NPS净推荐值 | 甲方愿意推荐平台的程度 | ≥ +30 | NPS 标准问卷 |
| 乙方NPS净推荐值 | 乙方愿意推荐平台的程度 | ≥ +40 | NPS 标准问卷 |
| 需求输入完成率 | 开始输入需求到成功发布的完成比率 | ≥ 80% | 漏斗转化分析 |
| 追问轮次 | 需求补全所需的平均对话轮次 | ≤ 4 轮 | 对话系统日志 |

#### 维度四：平台增长（权重 20%）

| 指标 | 定义 | 目标值 | 测量方式 |
|---|---|---|---|
| 双边用户留存率 | 甲/乙方 30 日留存 | ≥ 50% | 活跃用户追踪 |
| GMV 月增速 | 平台月度交易总额增长率 | ≥ 20% (MoM) | 财务报表 |
| 乙方平均月收入 | 活跃乙方通过平台的月均收入 | ≥ ¥5,000 (Phase 1) | 结算系统统计 |
| 重复交易率 | 甲乙方再次通过平台交易的比率 | ≥ 30% | 系统追踪 |

### 6.2 评测方法

#### 离线评测（Phase 1 核心）

1. **需求解析评测**
   - 构建 100 条标注真实场景需求（含行业/技能/预算/目标标注）
   - 评测指标：结构化准确性（字段级 F1 Score）、追问合理性（人工评级）

2. **匹配排序评测**
   - 构建 50 组历史交易匹配对（甲乙方配对 + 交易结果标签）
   - 评测指标：Recall@5、Precision@5、NDCG@5
   - 对比基线：关键词匹配（BM25）、纯向量语义搜索

3. **推荐理由评测**
   - 对 30 条 AI 生成的推荐理由进行人工评判
   - 评测指标：理由可信度（1-5 分）、证据充分性（引用数量）

#### 在线评测（上线后）

1. **A/B 测试**：对比 AI 匹配 vs 关键词搜索的效果差异
2. **用户行为分析**：埋点追踪完整用户旅程漏斗
3. **定期满意度调查**：每季度 NPS 和用户体验调研

### 6.3 晋级门槛与里程碑

## 九、总结

SuperOPCHub 不是一个传统的自由职业者平台。它的差异化在于——

- **不靠关键词匹配**，而是**语义理解 + 多维融合匹配**
- **不靠乙方自述**，而是**AI 解析真实作品/交易记录生成画像**
- **不靠简单列表**，而是**可解释的 AI 推荐理由**
- **不只做撮合**，而是**全链路 Agent 自动化（需求 → 匹配 → 签约 → 结算）**

在这份 Specs 对应的应用场景中，SuperOPCHub 拆掉的是**机会墙**——让每一个 OPC 不再把最聪明的时间浪费在"被找到"上，而是专注创造。当组织墙（OPC）、地理墙（数字游民）、移动墙（L4）、机会墙（SuperOPCHub）相继消失——

**剩下的就是自由。**
